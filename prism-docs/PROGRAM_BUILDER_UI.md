# Prism Platform — Program Builder UI

## Overview

The Program Builder allows administrators to design operational programs without writing code.

Programs include:

audits
assessments
competitions
inspections
surveys

---

## Builder Workflow

Step 1 — Create Program
Step 2 — Configure Program Settings
Step 3 — Create Sections
Step 4 — Add Questions
Step 5 — Configure Scoring
Step 6 — Assign Permissions
Step 7 — Publish

---

## Program Settings

Fields:

Program Name
Program Type
Department
Scoring Enabled
Offline Enabled
Image Upload Enabled
Geo-location Enabled
Signature Enabled

---

## Section Builder

Admins can create sections.

Example:

Cleanliness
Equipment
Beverage Quality

Each section can contain multiple questions.

---

## Question Builder

Admins can add questions to sections.

Question options include:

Text Input
Number Input
Yes/No
Multiple Choice
Dropdown
Image Upload
Rating Scale

---

## Question Configuration

Options include:

Required
Weight
Allow Comments
Allow Images
Enable Annotation
Conditional Logic

---

## Conditional Logic UI

Admins define rules.

Example:

If Question A = No → Show Question B

This is implemented using rule configuration.

---

## Scoring Configuration

Admins define scoring weights.

Example:

Cleanliness = 30%
Equipment = 30%
Beverage Quality = 40%

Total score automatically calculated.

---

## Preview Mode

Admins can preview programs before publishing.

Preview simulates the full submission process.

---

## Version Control

Programs support versioning.

Example:

QA Audit v1
QA Audit v2

Old submissions remain linked to original program versions.

---

## Publishing

Programs can be:

draft
active
archived

Only active programs can accept submissions.
